# Feedback Session Skill

A structured approach to giving and receiving feedback, based on McKinsey's bi-weekly feedback system.

## Core Principles

- **Two-way**: Both parties give feedback to each other equally
- **Bi-weekly cadence**: Regular, small fixes — not annual surprise bombs
- **Separate from performance review**: No salary or promotion talk here
- **Focus on working style only**: Not technical skills or domain expertise
- **No defending**: Recipient listens silently; no "but...", "because..." allowed

## How to Use This Skill

When the user wants to give feedback to someone, follow this process:

### Step 1: Gather Context

Ask the user:
1. **Who** are you giving feedback to? (role, seniority relative to you)
2. **What's the relationship**? (peer, direct report, manager)
3. **What period** does this cover?
4. **Any specific incidents** or patterns you've observed?

### Step 2: Structure the Feedback

Generate feedback in two parts:

#### Part A — What You Appreciate
Start with genuine positives. Format:
> "I enjoy working with you because [specific behavior/trait]..."

- Be specific, not generic ("you're great" is useless)
- Reference real moments or patterns
- 2–3 points max

#### Part B — What Can Improve
Areas for growth. Format:
> "You could improve how we work together by [specific behavior change]..."

- Focus on **behaviors and interactions**, not personality
- Make it actionable — describe what "better" looks like
- 2–3 points max; don't overwhelm

### Step 3: Output Format

Produce a clean feedback note the user can bring to the session:

```
FEEDBACK FOR: [Name/Role]
PERIOD: [Timeframe]
DATE: [Date]

✅ WHAT I APPRECIATE
• [Point 1 — specific]
• [Point 2 — specific]

🔧 WHAT COULD IMPROVE
• [Point 1 — specific behavior + what better looks like]
• [Point 2 — specific behavior + what better looks like]

SESSION REMINDER:
- Flip a coin to decide who goes first
- Listener stays silent — no defending, no explaining
- If you want to respond, save it for next session
```

## Rules for Good Feedback

| ✅ Do | ❌ Don't |
|---|---|
| "When you assign tasks at 5pm, I can't plan my day" | "You're disorganized" |
| "I appreciate that you always reply quickly" | "You're nice" |
| "It helps when you share blockers early" | Talk about technical incompetence |
| Focus on one or two key things | Dump everything at once |

## Tone Guidance

- **Peer feedback**: Direct and equal; no softening needed
- **Feedback to manager**: Frame as "what helps me do my best work"
- **Feedback to direct report**: Coaching tone; emphasize growth, not judgment

## Quick Mode

If the user says something like "quick feedback for X", skip the questions and ask only:
> "What's one thing they do well, and one thing that creates friction for you?"

Then generate a concise feedback note from their answer.
